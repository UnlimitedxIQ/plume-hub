# Frontend Design

## Purpose
Helps build polished, production-grade frontend components and pages with strong visual design — avoiding the generic, samey look of default AI-generated UI. Use when implementing interfaces that need to feel intentional, branded, and distinctive.

## When to use
- Building a marketing site or product UI that needs to feel premium
- Translating a design concept into a working component
- Replacing a generic-looking prototype with something distinctive
- Picking type, color, spacing, and motion for a new design system
- Polishing details after an MVP UI is functionally complete

## Approach
- Establish a strong type scale and color palette before any layout
- Use real content during design — placeholder text hides problems
- Add intentional motion and transitions, not just defaults
- Pay obsessive attention to spacing, alignment, and visual rhythm
- Reference 5-10 real-world examples for every component you build
