# MVP Builder

## Purpose
Guides founders through building a real MVP that tests demand without wasting months on premature engineering. Based on the manual-then-automated-then-productized progression popularized by minimalist entrepreneurship.

## When to use
- Validating a startup idea before writing real code
- The user is overbuilding and wants to scope down
- Choosing between no-code, low-code, and code-first approaches
- Designing the smallest possible test of demand
- Moving from manual delivery to a self-serve product

## Approach
- Start with the manual version — deliver the outcome by hand
- Charge real money before automating anything
- Automate only the steps that scale poorly
- Keep the tech stack as boring as possible
- Build the productized version only after demand is proven
